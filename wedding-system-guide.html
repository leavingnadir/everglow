🗄 Database: Neon PostgreSQL — Full Explanation

Option A vs Option B: Which approach to use?

✅ Recommended

One Shared Neon Database

  • All 6 members use the same connection string
  • All tables live in one database
  • Foreign keys work across components (e.g., bookings link to users)
  • No data sync issues between members
  • Neon free tier: 1 database is more than enough
  • Realistic team workflow
⚠️ Not Recommended

Separate Neon per Member

  • Member 3 (Bookings) can't query Member 1's users table
  • Member 4 (Payments) can't link to Member 3's bookings
  • Cross-component features break completely
  • 6× the setup and configuration
  • Neon free tier limit hit easily
  • Not how real systems work

How One Neon Database Works for the Whole Team

1
Team Lead creates ONE Neon project

Go to neon.tech → Create Account → New Project → Name it wedding-planning-db → Choose PostgreSQL region closest to your team.

2
Share the connection string with all 6 members

Neon dashboard → Connection Details → copy the DATABASE_URL. It looks like:

postgresql://username:password@ep-cool-name-12345.us-east-2.aws.neon.tech/wedding_db?sslmode=require

Share this via your group chat / shared document (keep it private — don't push to GitHub).

3
Each member stores it in their local .env file

In the project root, create a file called .env (never commit this to Git):

DATABASE_URL=postgresql://username:password@ep-xxx.neon.tech/wedding_db?sslmode=require
DB_HOST=ep-xxx.neon.tech
DB_PORT=5432
DB_NAME=wedding_db
DB_USER=your_username
DB_PASSWORD=your_password
4
One member runs the SQL schema once to create all tables

The team creates a shared schema.sql file with all 6 members' tables. Run it once in Neon SQL Editor or with a migration tool.

Complete Database Schema (All Members)

-- ══════════════════════════════════════════════════════════
-- WEDDING PLANNING SYSTEM — Full Schema
-- Run this ONCE in Neon SQL Editor
-- ══════════════════════════════════════════════════════════

-- MEMBER 1: User Management
CREATE TABLE users (
  user_id    SERIAL PRIMARY KEY,
  name       VARCHAR(100) NOT NULL,
  email      VARCHAR(150) UNIQUE NOT NULL,
  password   VARCHAR(255) NOT NULL,
  role       VARCHAR(20) DEFAULT 'CLIENT',  -- CLIENT or ADMIN
  phone      VARCHAR(20),
  wedding_date DATE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- MEMBER 2: Vendor Management
CREATE TABLE vendors (
  vendor_id    SERIAL PRIMARY KEY,
  name         VARCHAR(150) NOT NULL,
  category     VARCHAR(50),  -- PHOTOGRAPHER, CATERER, DECORATOR etc
  price        DECIMAL(10,2),
  availability BOOLEAN DEFAULT TRUE,
  rating       DECIMAL(3,2),
  contact      VARCHAR(100)
);

-- MEMBER 3: Booking Management
CREATE TABLE bookings (
  booking_id  SERIAL PRIMARY KEY,
  user_id     INT REFERENCES users(user_id),
  vendor_id   INT REFERENCES vendors(vendor_id),
  event_date  DATE NOT NULL,
  status      VARCHAR(20) DEFAULT 'PENDING',  -- PENDING, CONFIRMED, CANCELLED
  total_price DECIMAL(10,2),
  created_at  TIMESTAMP DEFAULT NOW()
);

-- MEMBER 4: Payment & Invoice
CREATE TABLE payments (
  payment_id     SERIAL PRIMARY KEY,
  booking_id     INT REFERENCES bookings(booking_id),
  amount         DECIMAL(10,2) NOT NULL,
  payment_method VARCHAR(30),  -- ONLINE, CASH
  status         VARCHAR(20) DEFAULT 'PENDING',
  paid_at        TIMESTAMP
);
CREATE TABLE invoices (
  invoice_id   SERIAL PRIMARY KEY,
  booking_id   INT REFERENCES bookings(booking_id),
  total_amount DECIMAL(10,2),
  issued_date  DATE DEFAULT CURRENT_DATE
);

-- MEMBER 5: Package Management
CREATE TABLE packages (
  package_id  SERIAL PRIMARY KEY,
  name        VARCHAR(100),  -- Silver, Gold, Platinum
  price       DECIMAL(10,2),
  description TEXT
);
CREATE TABLE package_services (
  package_id INT REFERENCES packages(package_id),
  vendor_id  INT REFERENCES vendors(vendor_id),
  PRIMARY KEY(package_id, vendor_id)
);

-- MEMBER 6: Review & Feedback
CREATE TABLE reviews (
  review_id  SERIAL PRIMARY KEY,
  user_id    INT REFERENCES users(user_id),
  vendor_id  INT REFERENCES vendors(vendor_id),
  rating     INT CHECK(rating BETWEEN 1 AND 5),
  comment    TEXT,
  date       TIMESTAMP DEFAULT NOW(),
  verified   BOOLEAN DEFAULT FALSE
);
✅ Because all tables are in one database, Member 3 can JOIN with users and vendors tables, Member 6 can JOIN reviews with users, and Member 4 can link payments to bookings. This is how a real system works.

OOP Concepts in Code — Example (Member 1: Users)

// ── ENCAPSULATION ──────────────────────────────────────────
// User.java (Base Class) — private fields + getters/setters
public abstract class User {
    private Long userId;
    private String name;
    private String email;
    private String password;

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    // ... other getters/setters
}

// ── INHERITANCE ────────────────────────────────────────────
public class ClientUser extends User {
    private LocalDate weddingDate;
    private String phone;
    // Client-specific methods
}

public class AdminUser extends User {
    private String adminCode;
    // Admin-specific methods
}

// ── POLYMORPHISM ───────────────────────────────────────────
public interface UserService {
    String getDashboardView();
}
// ClientUserServiceImpl and AdminUserServiceImpl each implement
// getDashboardView() differently — polymorphism in action

// ── REST CONTROLLER ────────────────────────────────────────
@RestController
@RequestMapping("/api/users")
public class UserController {
    @Autowired
    private UserService userService;

    @PostMapping("/register")
    public ResponseEntity register(@RequestBody UserDTO dto) {
        return ResponseEntity.ok(userService.register(dto));
    }

    @GetMapping("/{id}")
    public ResponseEntity getUser(@PathVariable Long id) {
        return ResponseEntity.ok(userService.findById(id));
    }
}

Frontend calls Backend — CORS configuration

// backend/config/CorsConfig.java — allows frontend to call backend
@Configuration
public class CorsConfig {
    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                    .allowedOrigins("http://localhost:5173")  // React dev server
                    .allowedMethods("GET", "POST", "PUT", "DELETE");
            }
        };
    }
}
// frontend/src/services/api.js — axios base config
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:8080/api',
});

// Attach JWT token to every request automatically
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

export default api;
